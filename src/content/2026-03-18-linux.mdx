# **Linux: el sistema operativo que cambió el mundo**

Alguna vez te preguntaste quien creo Linux? Quien comenzó a desarrollar un sistema operativo Open Source? O mismo como mier** se programa
un sistema operativo? Bueno, vayamos por partes para entender algunos conceptos básicos..

## **¿Qué es un sistema operativo?**

Retomando un poco los apuntes de la materia Sistemas operativos me encontré con la siguiente definition:

> El software que maneja los recursos de la computadora responsable de hacer a los programas fáciles de “correr”, permitiéndoles compartir memoria,
establecer conexiones con otros dispositivos, etc.

Si estas leyendo esto desde un celular, quizás estas bajo un sistema operativo Android o iOS, si estas desde una computadora, quizás estas bajo un Windows, MacOS o Linux. Básicamente, esos sistemas lo que le permiten es a tu dispositivo interactuar con el hardware y ejecutar programas. El hardware
seria como el cerebro del dispositivo, justamente empieza con HARD- porque es la parte física (dura). Involucra a la memoria, el procesador, el disco rígido, etc. Todos esos elementos de alguna manera vamos a querer utilizarlos a nuestro favor, ya sea para jugar algún juego online, escribir mails, programar, editar videos.. Y ahi es donde entra en juego el Sistema Operativo, que se va a encargar de manejar esos recursos y permitirnos interactuar con ellos a traves de una interfaz gráfica o de comandos.

Con esta imagen podemos entender un poco mejor como funciona un sistema operativo:

![Sistema Operativo](/images/operating-system.png)

En la imagen vemos cómo el sistema operativo se encuentra en el medio, entre el hardware y el software. El hardware de cada computadora puede ser distinto: varía según el modelo, la marca o los componentes que tenga. Ahora imaginemos que para cada tipo de hardware hubiera que programar software diferente para poder comunicarse con esos componentes específicos. Sería un caos, ¿no?

Antes del auge de los sistemas operativos modernos, el software estaba mucho más ligado al hardware. Esto hacía que fuera muy difícil para los programadores crear programas que funcionaran en distintas computadoras.

En esa época se utilizaba algo llamado **[lenguaje ensamblador](https://es.wikipedia.org/wiki/Lenguaje_ensamblador)**, que es un conjunto de instrucciones de muy bajo nivel específicas para cada arquitectura de procesador. Este lenguaje proporciona una representación simbólica del código máquina, es decir, una forma más legible para los programadores de escribir las instrucciones binarias que entiende la computadora (recordemos que la máquina solo interpreta 0 y 1). Eso significa que Cada arquitectura de procesador tiene su propio lenguaje ensamblador. Como consecuencia, hacer que una misma aplicación funcionara en más de una computadora muchas veces implicaba reescribir el programa para cada arquitectura, nuevamente en ensamblador.

## **¿Quién creo Linux?**

### Historia de UNIX, su antecesor

En 1969 nace la primer versión de Unix, un sistema operativo desarrollado por un grupo de programadores en los laboratorios Bell de AT&T, desarrollado por Ken Thompson, Dennis Ritchie y otros. Surge tras el fracaso de un fracaso de un proyecto anterior llamado MULTICS, un sistema operativo muy avanzado que termino siendo demasiado complejo. Thompson decide crear un sistema **más simple y elegante**, y así nace la primer versión de Unix. 

Entre 1971 y 1973, Dennis Ritchie crea [C](https://es.wikipedia.org/wiki/C_(lenguaje_de_programaci%C3%B3n)), un lenguaje de alto nivel orientado a la implementación de sistemas operativos. 
En 1973, Unix es reescrito en C, lo que facilita su portabilidad a diferentes arquitecturas de hardware. Esto permitió que Unix se convirtiera en un sistema operativo muy popular en universidades y empresas, y sentó las bases para el desarrollo de muchos otros sistemas operativos.

Las **[bases de Unix](https://es.wikipedia.org/wiki/Filosof%C3%ADa_de_Unix#Las_17_Reglas_de_Unix_de_Eric_Raymond)** se convirtieron en un estándar para el desarrollo de sistemas operativos y su influencia se puede ver en muchos sistemas operativos modernos, incluyendo Linux. De hecho, Linux se considera un sistema operativo "Unix-like" porque sigue los principios y la filosofía de diseño de Unix.

1. ***Todo es un archivo***: Todo se representa como un archivo, incluyendo dispositivos de hardware, procesos y recursos del sistema. Esto simplifica la interacción con el sistema operativo y permite una mayor flexibilidad. Ejemplo:

| Recurso                 | Archivo                 |
|-------------------------|-------------------------|
| Teclado                 | `/dev/tty`              |
| Disco duro              | `/dev/sda`              |
| Impresora               | `/dev/lp0`              |
| Información del sistema | `/proc`                 |
| Archivo normal          | `/home/user/documento.txt` |

2. ***Modularidad = "Haz una sola cosa y hazla bien"***: Enfocarse en herramientas pequeñas en lugar de programas grandes y complejos que hacen todo (KISS = Keep It Simple Stupid). Unix provee cientos de "miny tools"
como `grep`, `ls`, `sort`, entre otras. 

3. ***Escribe programas que trabajen juntos***: El output de cada programa debería de convertirse en el input de otro. Unix introdujo el concepto de [`pipe`](https://www.geeksforgeeks.org/linux-unix/piping-in-unix-or-linux/), que permite tomar el output de un programa y usarlo como input de otro programa,
lo que facilita la creación de flujos de trabajo complejos utilizando herramientas simples. 

4. ***Escribe programas que manejen stream de texto, porque esa es la interfaz universal***: Los programas deben estar diseñados para funcionar con entrada y salida de texto, que pueden ser fácilmente generados y consumidos por otros programas.

Obviamente hay unos pares mas de conceptos y principios que Unix introdujo pero se puede decir que estos fueron los mas influyentes para el desarrollo de Linux.

### Minix ###

En el año 1987 Andrew Tanenbaum, un profesor holandés apasionado por la enseñanza y la informática se encontró ante los problemas prácticos y legales para 
enseñar sistemas operativos basándose en Unix, que en ese momento era el estándar, pero un código cerrado. Por eso Tanenbaum, decidió crear un sistema operativo pequeño, sencillo y abierto que
pudiera ser estudiado línea por línea en las aulas.

Para resolver esto, decidió crear su propio sistema: *MINIX* (Mini Unix).

![Minix](/images/minix.jpg)

Minix fue diseñado con un objetivo claro:
> ser un sistema operativo educativo, fácil de entender y de modificar por los estudiantes.

Sus principales características fueron:
- Implementaba la filosofía de UNIX (todo es un archivo, modularidad, etc.)
- Tenía un *microkernel*, lo que significa que el núcleo del sistema operativo era muy pequeño y la mayoría de las funciones 
se ejecutaban en espacio de usuario, lo que facilitaba su comprensión y modificación.
- Era lo suficientemente simple para ser estudiado en su totalidad por los estudiantes, con un código fuente de aproximadamente 12,000 líneas.
- Corría en computadoras personales con recursos limitados.

A diferencia de Unix, MINIX no estaba pensado para producción, sino para enseñar cómo funciona un sistema operativo por dentro.

#### Por qué fue importante MINIX?

MINIX desempeñó un papel fundamental en el desarrollo inicial de Linux, tanto de forma directa como indirecta. 
Linus Torvalds lo utilizó como punto de partida para escribir su propio núcleo de sistema operativo en 1991. 
Si bien Torvalds no copió directamente el código de MINIX en Linux, este proporcionó un modelo de cómo estructurar un sistema operativo tipo Unix 
en ordenadores personales económicos.

### Linux: “just a hobby” ###

En 1991, Linus Torvalds publicó un mensaje en un grupo de usuarios de MINIX que hoy es histórico.

En ese mensaje escribió:
![Linux](/images/linus-torvald-first-linux-email.webp)

La imagen muestra ese primer email, enviado a la comunidad de MINIX, donde Linus anunciaba su proyecto. El estaba utilizando MINIX 
para aprender sobre sistemas operativos, pero quería crear algo nuevo, algo que fuera completamente libre y abierto. Entonces decidió hacer
algo bastante ambicioso: escribir un sistema operativo desde cero, inspirado en Unix pero sin copiar el código de MINIX.

#### El primer paso

Linus comenzó escribiendo el **kernel**, es decir, el núcleo del sistema operativo encargado de:

- manejar la memoria  
- controlar procesos  
- interactuar con el hardware  

Al principio, Linux:

- corría solo en procesadores Intel 80386  
- usaba herramientas de MINIX para desarrollarse  
- incluso utilizaba el sistema de archivos de MINIX  

Era algo muy básico, pero funcional. Por eso, en lugar de guardarse el proyecto, Linus tomó una decisión clave:

> publicó el código en internet y permitió que cualquiera pudiera usarlo, modificarlo y mejorarlo.

Ese fue el punto de inflexión ya que desarrolladores de todo el mundo empezaron a colaborar:

- agregando funcionalidades  
- corrigiendo errores  
- adaptándolo a nuevas arquitecturas  

Linux dejó de ser un proyecto individual y se convirtió en un proyecto colaborativo global. 

Linux por sí solo es solo el **kernel**, es decir, el núcleo del sistema operativo.  
Para que sea un sistema completo, se combina con herramientas del proyecto GNU, iniciado por [Richard Stallman](https://es.wikipedia.org/wiki/Richard_Stallman).
Por eso muchas veces se habla de **GNU/Linux**.

Entonces ¿Por qué Linux fue tan importante? Porque tomó las ideas de UNIX y las llevó más lejos:

- es libre y open source  
- es portable (corre en múltiples arquitecturas)  
- es modular y flexible  
- puede usarse desde una computadora personal hasta un supercomputador  

Lo que empezó como un proyecto personal terminó siendo:

- la base de servidores en internet  
- el corazón de Android  
- el sistema dominante en cloud y supercomputadoras  

En otras palabras, Unix sentó las bases, MINIX enseñó cómo funcionaban… y Linux las llevó al mundo real.
