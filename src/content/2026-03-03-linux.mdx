# **Linux: el sistema operativo que cambió el mundo**

Alguna vez te preguntaste quien creo Linux? Quien comenzo a desarrollar un sistema operativo Open Source? O mismo como mier** se programa
un sistema operativo? Bueno, vayamos por partes para entender algunos conceptos básicos..

## **¿Qué es un sistema operativo?**

Retomando un poco los apuntes de la materia Sitemas operativos me encontre con la siguiente definicion:

> El software que maneja los recursos de la computadora responsable de hacer a los programas fáciles de “correr”, permitiendoles compartir memoria,
establecer conexiones con otros dispositivos, etc.

Si estas leyendo esto desde un celular, quizas estas bajo un sistema operativo Android o iOS, si estas desde una computadora, quizas estas bajo un Windows, MacOS o Linux. Basicamente, esos sistemas lo que le permiten es a tu dispositivo interactuar con el hardware y ejecutar programas. El hardware
seria como el cerebro del dispositivo, justamente empieza con HARD- porque es la parte fisica (dura). Involucra a la memoria, el procesador, el disco rigido, etc. Todos esos elementos de alguna manera vamos a querer utilizarlos a nuestro favor, ya sea para jugar algun juego online, escribir mails, programar, editar videos.. Y ahi es donde entra en juego el Sistema Operativo, que se va a encargar de manejar esos recursos y permitirnos interactuar con ellos a traves de una interfaz grafica o de comandos.

En la imagen vemos cómo el sistema operativo se encuentra en el medio, entre el hardware y el software. El hardware de cada computadora puede ser distinto: varía según el modelo, la marca o los componentes que tenga. Ahora imaginemos que para cada tipo de hardware hubiera que programar software diferente para poder comunicarse con esos componentes específicos. Sería un caos, ¿no?
Antes del auge de los sistemas operativos modernos, el software estaba mucho más ligado al hardware. Esto hacía que fuera muy difícil para los programadores crear programas que funcionaran en distintas computadoras. En esa época se utilizaba algo llamado lenguaje ensamblador, que es un conjunto de instrucciones de muy bajo nivel específicas para cada arquitectura de procesador. Eso significa que cada tipo de CPU tiene su propio lenguaje ensamblador. Como consecuencia, hacer que una misma aplicación funcionara en más de una computadora muchas veces implicaba reescribir el programa para cada arquitectura, nuevamente en ensamblador.
